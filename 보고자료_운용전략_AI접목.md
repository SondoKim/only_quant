# 운용전략에 AI 모델 접목 — 채권부문 현업 개선 사례

**발표 주제 (3대 축 中 ②번)**
운용전략에 AI 모델(머신러닝 + 퀀트모델)을 접목하여, 채권부문이 보유한 **글로벌 매크로 도메인 지식**을 시스템화하고 운용 의사결정의 **속도·일관성·확장성**을 개선한 사례

---

## 한 줄 요약

> **"펀드매니저의 머릿속 매매 논리를 AI 에이전트가 24시간 자동으로 수만 개 검증·집계하여, 매일 아침 매매 신호로 받아보는 체계."**

---

## 1. 배경 — 왜 운용에 AI가 필요했나

| 기존 방식의 한계 | 결과 |
|---|---|
| 매니저별 경험·직관에 의존 | 일관성 부족, 인적 의존도 ↑ |
| 글로벌 다자산(미국/영국/호주/한국/독일/일본 금리 + 주요 FX) 동시 분석 곤란 | 기회 누락, 분산효과 미흡 |
| 매매 아이디어 검증에 수일~수주 소요 | 시장 변화 대응 지연 |
| 백테스트가 엑셀·수기 기반 | 재현성·확장성 한계 |

**→ 채권부문의 도메인 지식을 코드화하고, AI가 대신 검증·운용하는 체계 구축**

---

## 2. 시스템 개요 — 한 장으로 보는 구조

```
   [Bloomberg 데이터]                         (8개국 금리 + 5개 통화 + 지수)
            │
            ▼
   ① AI 전략 발굴 에이전트   ─── 매니저 도메인 지식을 룰로 코드화
            │                  (모멘텀 / 평균회귀 / 캐리 / 스프레드 / 커브 …)
            ▼
   ② 퀀트 백테스트 엔진      ─── 수만 개 조합을 동시 검증
            │                  (벡터화 연산 + 멀티프로세싱)
            ▼
   ③ 전략 공장 (Factory)     ─── 통과한 전략만 월별 DB로 보관
            │                  (Sharpe·Sortino·거래수 임계 통과 시 저장)
            ▼
   ④ AI 포트폴리오 선택      ─── 상관관계 낮은 전략만 골라
            │                  성과 가중으로 자산별 포지션 산출
            ▼
   ⑤ 매일 아침 매매 신호     ─── 자산별 ±1 포지션 + 사이즈
```

---

## 3. 3대 핵심 개선사례

### ① 도메인 지식 기반 (Domain-Driven AI)

**핵심:** AI가 "혼자 학습"한 것이 아니라, **채권 운용 전문가의 매매 논리를 그대로 룰로 옮긴 후** AI가 이를 대규모로 검증·운용하는 구조.

- **운용역의 매매 논리 → 코드화**
  - 모멘텀 (이평 교차, MACD, 돌파)
  - 평균회귀 (Z-score, 볼린저, RSI 극단)
  - **금리 커브 / 크로스컨트리 스프레드 / 캐리** 등 채권 고유 전략
  - Lead-Lag, 변동성 돌파, 상대강도(RS) 등 매크로 전략
- **자산 유니버스도 채권부문 전문영역 중심으로 설계**
  - Rates: US 2Y/10Y, UK·DE·FR·IT·JP·AU·KR
  - FX: EUR / GBP / JPY / AUD / KRW
- **"블랙박스" AI가 아닌 "화이트박스" 룰 기반 AI**
  - 모든 전략의 진입/청산 조건이 사람이 읽을 수 있는 룰로 저장
  - → 컴플라이언스·리스크관리·임원 보고에 그대로 활용 가능

> **시사점:** AI 도입의 출발점은 알고리즘이 아니라 **도메인 지식의 정형화**. 우리의 강점인 채권 매크로 지식을 자산화하는 데 AI를 활용한 것.

---

### ② 실효적 AI 활용 (Practical AI, Not Buzzword)

**핵심:** "AI를 썼다"가 아니라 **실제 운용 의사결정의 품질을 측정 가능하게 개선**한 부분.

- **AI 에이전트의 역할 (사람이 못 하는 일을 대신)**
  1. **수만 개 전략 동시 백테스트**: 매니저 1명이 수일 걸릴 검증을 분 단위로 수행
  2. **상관관계 기반 포트폴리오 자동 구성**: 비슷한 전략 중복 제거 → 진정한 분산 확보
  3. **성과 가중 집계**: 최근 잘 작동하는 전략에 자동으로 더 큰 비중 부여 (적응적)
  4. **Walk-Forward 구조**: 미래 데이터를 보지 않고 발굴 → 실거래 환경 그대로 시뮬레이션 (Look-ahead bias 차단)
- **"머신러닝 = 예측"이 아닌, "머신러닝 = 검증·선택"**
  - 가격 예측 모델이 아닌, **수많은 매매 가설 중 통계적으로 유효한 전략을 선별**하는 방식
  - 과적합(Overfitting) 위험을 구조적으로 통제 (월별 폴더 분리, 임계값 통과 검증)
- **퀀트 모델과 AI의 결합**
  - 퀀트 모델(룰 기반) = 매매 논리 자체
  - AI/ML = 어떤 룰의 어떤 파라미터가 현재 시장에 적합한지 선별하는 메타 레이어

> **시사점:** AI는 매니저를 대체하는 것이 아니라, 매니저가 **할 수 없는 규모와 일관성**을 보완하는 도구.

---

### ③ 워크플로우 및 생산성 향상 (Workflow & Productivity)

**핵심:** 리서치 → 검증 → 운용으로 이어지는 사이클을 **수동 → 자동**으로 전환.

| 구분 | Before | After |
|---|---|---|
| 매매 아이디어 검증 | 수일~수주 (수기 백테스트) | 수 분 (자동 백테스트) |
| 검증 가능한 전략 수 | 매니저당 수십 개 | 시스템당 **수만 개** |
| 신호 산출 | 매니저 개별 판단 | **매일 아침 자동 산출** |
| 데이터 일관성 | 매니저별 상이 | 전사 동일 (Bloomberg → DB) |
| 재현성 | 낮음 | 모든 전략·신호 DB 보관 → 100% 재현 |
| 다자산 동시 운용 | 인적 한계 | 13개 금리 + 5개 FX 동시 커버 |

- **운용역의 시간을 "단순 검증"에서 "전략 설계·리스크 판단"으로 재배치**
- **월말 자동 갱신 사이클**: 매월 신규 데이터로 전략 풀 자동 재구성 → 시장 변화 자동 반영
- **데이터·전략·신호 모두 DB(SQLite) 통합 관리**: 컴플라이언스·감리 대응 즉시 가능

> **시사점:** AI가 운용역의 "잡일"을 가져가고, 운용역은 **고차원적 판단(리스크 한도, 전략 설계, 시장 해석)**에 집중.

---

## 4. 정량 효과 (예시 지표)

> 발표 시 실제 수치로 교체 권장

- **검증 가능 전략 수**: 수십 개 → **수만 개** (×1,000 이상)
- **신호 산출 주기**: 주 단위 → **일 단위 자동화**
- **커버 자산 수**: 일부 자산 → **8개국 금리 + 5개 FX**
- **백테스트 소요 시간**: 수일 → **분 단위**
- **재현성**: 정성적 → **100% 재현 가능 (DB 보관)**

---

## 5. 향후 발전 방향

1. **AI 기반 시장 국면(Regime) 인식과 결합** → 발표 ③번 주제와 연계
   - 변동성·상관관계 레짐 변화 시 전략 풀 자동 재구성
2. **글로벌 시황·리서치 요약 AI와 결합** → 발표 ①번 주제와 연계
   - 시황 요약 + 전략 신호를 동일 리포트에서 일원화
3. **리스크 관리 고도화**
   - 역변동성 사이징, 스트레스 테스트, 동적 레버리지

---

## 6. 핵심 메시지 (발표 마무리용)

> 1. 우리는 **AI를 위한 AI가 아닌, 도메인 지식의 자산화 도구로서 AI**를 도입했다.
> 2. 수만 개 전략을 24시간 검증하는 **AI 에이전트**가 운용역을 보좌한다.
> 3. 그 결과 채권부문의 **운용 일관성·속도·확장성**이 구조적으로 개선되었다.

- ① ② ③ 슬라이드 구조를 그대로 사용하되, 각 슬라이드당 30~60초 분량으로 압축
- "AI 에이전트", "퀀트 모델", "백테스트", "포트폴리오 자동 구성" 정도의 용어만 사용 (Sharpe, Walk-forward 등은 부연 설명 시에만)
- 시스템 구조도(섹션 2)는 **한 장으로 전체를 보여주는 핵심 슬라이드**로 활용
